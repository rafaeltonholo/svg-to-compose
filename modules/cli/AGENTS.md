# svg-to-compose-cli - Agent Context

## Overview

Standalone CLI module for the SVG-to-Compose tool. This is a Gradle
composite build (`includeBuild`) that depends on the core `svg-to-compose`
library and produces native executables for macOS and Linux, a Windows
executable, and a JVM fat JAR for platforms without native support.

## Source Structure

```
modules/cli/
  build.gradle.kts          # KMP config, native executable, Shadow fat JAR
  settings.gradle.kts       # Composite build wiring to core + build-logic
  src/
    commonMain/
      kotlin/
        dev/tonholo/s2c/cli/
          Main.kt            # Entry point (shared across all targets)
          inject/
            CliGraph.kt      # Metro DI graph (AppScope)
          runtime/
            Client.kt        # Clikt command, CLI argument definitions
            CliConfig.kt     # Mutable S2cConfig implementation
```

## Dependencies

- `svg-to-compose` (core) - parsing/conversion logic
- Clikt 5.1.0 - CLI framework
- Clikt Markdown - help formatting
- Okio - file system operations
- Shadow - JVM fat JAR packaging

## Key Concepts

- **Client** (`CliktCommand`): Parses CLI arguments and delegates to
  the core `Processor` via Metro DI.
- **CliConfig**: Mutable `S2cConfig` created with defaults, updated in
  `Client.run()` after Clikt parses flags. Components holding a reference
  (e.g. `CommonLogger`) see updated values immediately.
- **CliGraph**: `@DependencyGraph(AppScope)` that receives core bindings
  from `SvgToComposeBindings` (contributed `@BindingContainer`) and adds
  CLI-specific providers (FileSystem, Logger).
- **BuildConfig**: Generated by the `buildconfig` plugin under
  `dev.tonholo.s2c.cli.config`. Provides VERSION for `--version`.

## Build Output

**Native binaries:**
```
modules/cli/build/bin/{platform}/{debug,release}Executable/s2c.{kexe,exe}
```
Platforms: `macosArm64`, `linuxX64`, `mingwX64`.

**JVM fat JAR:**
```
modules/cli/build/libs/svg-to-compose-cli-*-all.jar
```
Runnable via `java -jar`. Built with Shadow plugin and `minimize()`.

## Build Commands

```bash
# Native (platform-specific)
./gradlew -p modules/cli buildDebugMacosarm64
./gradlew -p modules/cli buildReleaseMacosarm64

# JVM fat JAR
./gradlew -p modules/cli shadowJar

# All targets
./gradlew -p modules/cli build
```

## Conventions

- Package: `dev.tonholo.s2c.cli`
- Source in `commonMain` (shared between native and JVM)
- Uses `cliLibs` version catalog (`gradle/libs.cli.versions.toml`) for
  CLI-specific deps, plus `libs` catalog for shared deps (okio, shadow)
- Convention plugins from `build-logic/` apply automatically
